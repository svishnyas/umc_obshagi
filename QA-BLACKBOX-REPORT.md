# Отчёт чёрного ящика: УМЦобщаги

**Дата:** 2026-05-04  
**Метод:** разбор API и UI в репозитории + `eslint src` (без полного E2E в браузере).  
**Стек:** Next.js (App Router), NextAuth (credentials), Prisma (SQLite), TanStack Query.

## А. Регистрация и авторизация

| Сценарий из чеклиста | В проекте | Вердикт |
|----------------------|-----------|---------|
| Некорректный **email** | Email **не используется**. Вход: ник + пароль + **код общаги 6 цифр** (`src/auth.ts`, `RegisterForm`). | **N/A** — не баг, другая модель. |
| Короткий пароль | Клиент: ≥6 символов. Сервер: `registerSchema` `password.min(6)` (`api/auth/register`). | **Ожидаемо:** отклонение. |
| Спецсимволы / эмодзи в нике | Zod: `nickname` trim, 1–80 символов, **без запрета** на Unicode/эмодзи. | **Риск Low:** отображение через React — экранирование по умолчанию. |
| Дубликат ника в общаге | `nickname_dormId` unique → **409** «Такой ник уже занят…». | **Ок.** |
| «Забыл пароль» | Маршрутов / UI восстановления **нет**. | **Gap (Medium):** функция не реализована, не баг регрессии. |
| Logout / сессия | NextAuth JWT, `signOut`, `session.maxAge` в `auth.ts`. | **По коду ок**; ручная проверка cookie. |

### Замечание (Low)

**Название:** Регистрация: общее сообщение при ошибке Zod  

**Шаги:** POST `/api/auth/register` с телом, не проходящим схему (например пустой ник).  

**Ожидаемо:** Понятные ошибки по полям.  

**Факт:** `{ error: "Проверь поля формы" }` без детализации (`register/route.ts`).  

**Приоритет:** Low

## Б. Контент: посты

| Сценарий | Вердикт |
|----------|---------|
| Пустой пост | `POST /api/posts`: пустой `text` и нет фото → **400** «Добавь текст или фото». | **Ок.** |
| Огромный текст | `text.max(8000)`. | **Ок.** |
| Редактирование чужого поста | API **изменения текста поста** не найден; есть только **модерация вопросов** (`PATCH .../moderate`) с проверкой `isOwner` в БД. | **Ок** (нет лишнего PATCH для автора). |
| BBCode / Markdown | Текст выводится как `{post.text}` / `{c.text}` — **без** `dangerouslySetInnerHTML` (`PostCard.tsx`). Markdown не рендерится. | **XSS в разметке:** низкий риск классического HTML-injection в тексте поста. |
| HTML в поле | Сохранится как строка, на экране как текст. | **Ок.** |
| Тяжёлое изображение | `/api/upload`: max **5 МБ**, типы jpeg/png/webp/gif. | **Ок** для «50 МБ GIF» — отказ. |

### Замечание (Medium)

**Название:** Лента постов без пагинации и жёсткого лимита выборки  

**Шаги:** Авторизованный `GET /api/posts` при большой БД.  

**Ожидаемо:** Пагинация / «показать ещё».  

**Факт:** `findMany` без `take` (`posts/route.ts`).  

**Приоритет:** Medium (производительность и UX, не обязательно «сломает» сразу).
