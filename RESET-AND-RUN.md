# Сброс «с нуля» и запуск проекта (УМЦобщаги)

Краткая памятка для Windows. Все команды выполняй **в папке проекта** (где лежит `package.json`), в терминале **PowerShell** или **cmd**.

---

## Что должно быть установлено

- **Node.js** LTS (с сайта nodejs.org), чтобы в терминале работали команды `node` и `npm`.
- Если `npm` «не находится» — переоткрой терминал после установки Node или используй **«Node.js command prompt»**.

---

## Важно: что в проекте задумано так (не баги)

- **Нет email при регистрации** — только **ник + пароль + код общаги** (6 цифр). Вход по этим же данным.
- **Нет** сценария «забыл пароль», **нет** рассылки писем и **нет** push-уведомлений вне приложения. Уведомления есть **только внутри сайта** (колокольчик).
- **Нет** дерева комментариев (ответы на ответы) — комментарии **плоским списком** под постом.
- **Нет** бана пользователя и отдельной «жалобы» на пост — такой функциональности пока нет.

Если в ТЗ нужны email, пуши или модерация жалоб — это отдельные задачи на разработку.

---

## 1. Полный сброс данных и зависимостей (опционально)

Закрой **все** окна, где запущен `npm run dev`, и закрой Cursor/VS Code с этим проектом (так проще избежать блокировки файлов Prisma на Windows).

Удали:

| Что удалить | Зачем |
|-------------|--------|
| Папку `node_modules` | Чистая переустановка пакетов |
| Файл `package-lock.json` (если есть) | Жёсткий сброс lock-файла (не обязательно) |
| Файл `prisma/dev.db` | **Полный сброс базы** (все пользователи и посты пропадут) |
| Папку `public/uploads` кроме `.gitkeep` (по желанию) | Убрать загруженные картинки |

Про `.env`: обычно **не удаляй** — в нём секреты и `DATABASE_URL`. Если файла нет — скопируй:

```text
copy .env.example .env
```

(В PowerShell можно: `Copy-Item .env.example .env`)

---

## 2. Установка пакетов

```bash
npm install
```

После установки скрипт `postinstall` попытается выполнить `prisma generate`. Если в конце увидишь предупрошение про Prisma — не пугайся, сделай шаг 3.

---

## 3. База данных и Prisma Client (обязательно перед первым запуском)

Останови dev-сервер, если он был запущен. Затем **одной командой**:

```bash
npm run db:sync
```

Она:

1. Применяет схему к SQLite (`prisma db push`).
2. Несколько раз пытается выполнить `prisma generate`.

### Если вылезает EPERM / «operation not permitted» (Windows)

1. Закрой `npm run dev`.
2. Закрой Cursor с этим проектом **или** добавь в исключения антивируса папку  
   `node_modules\.prisma`
3. Снова выполни: `npm run db:sync`

---

## 4. Демо-данные (по желанию)

Чтобы в базе появились общаги, пользователи и посты для проверки:

```bash
npm run db:seed
```

Пароли и коды общаг смотри в `prisma/seed.ts` (у демо-пользователей пароль **`demo123`**, владелец — **«Владыка Форума»** с кодом общаги из сида).

---

## 5. Запуск сайта в режиме разработки

```bash
npm run dev
```

В браузере открой адрес из вывода терминала (часто **http://localhost:3000**). Если порт занят, Next.js может выбрать **3001** — смотри строку `Local:` в логе.

- Страница входа: **`/auth`**
- Лента: **`/feed`** (посты подгружаются страницами по 30, кнопка **«Показать ещё»** внизу ленты)
- Сквады комнат: **`/squads`**

---

## 6. Сборка для «как в проде» (по желанию)

```bash
npm run build
npm start
```

`build` снова вызовет `prisma generate` — к этому моменту EPERM на Windows лучше уже устранить.

---

## Частые проблемы

| Симптом | Что сделать |
|---------|-------------|
| Ошибки про таблицы / сквады | Снова `npm run db:sync` |
| «Нужна авторизация» / сессия | Проверь `.env`: `AUTH_SECRET`, `NEXTAUTH_URL` |
| Порт 3000 занят | Останови другой процесс или используй порт из лога Next |

---

## Минимальный путь без полного удаления `node_modules`

Если нужно только **обнулить базу**:

1. Останови dev-сервер.
2. Удали файл `prisma/dev.db`.
3. Выполни: `npm run db:sync`
4. При необходимости: `npm run db:seed`
5. `npm run dev`

На этом всё.
